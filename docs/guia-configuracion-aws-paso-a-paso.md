## 1. Crear cuenta AWS Free Tier

Si ya tienes cuenta, salta al paso 2.

1. Ve a [https://aws.amazon.com/free/](https://aws.amazon.com/free/)
2. Click **Create a Free Account**
3. Ingresa email, nombre de cuenta
4. Verificar email con el código que te envían
5. Ingresar datos de tarjeta de crédito/débito (AWS hace un cargo temporal de $1 USD que se revierte)
6. Verificar identidad por teléfono (SMS o llamada)
7. Seleccionar plan **Basic Support (Free)**

> **Free Tier incluye (12 meses):**
> - EC2: 750 horas/mes de t2.micro o t3.micro
> - S3: 5 GB de storage
> - RDS: 750 horas/mes de db.t3.micro
> - Elastic Beanstalk: gratis (solo pagas la EC2 subyacente)

## 2. Configurar MFA en la cuenta root

AWS te obliga a configurar autenticación de dos factores. Esto es lo que se ve al entrar por primera vez a IAM.

### Pasos:

1. Ve a [https://console.aws.amazon.com/iam/](https://console.aws.amazon.com/iam/) → **Security credentials**
2. En la sección "Assign MFA device" que aparece:
   - **Device name:** escribir `mi-celular`
   - **MFA device:** seleccionar **Authenticator app** (segunda opción)
   - Click **Next**
3. Te muestra un **código QR**
4. Abre **Google Authenticator** o **Microsoft Authenticator** en tu celular
5. Escanea el código QR
6. Ingresa **2 códigos consecutivos** que te da la app (espera a que cambie para el segundo)
7. Click **Assign MFA**

> **¿Por qué Authenticator app y no Passkey?**  
> Passkey requiere hardware específico (FIDO2). La app de autenticación funciona en cualquier celular y es igual de segura para nuestro propósito.

## 3. Crear usuario IAM para deploy

**NUNCA uses la cuenta root para deploy.** Crea un usuario específico con permisos limitados.

### Pasos:

1. Ve a [https://console.aws.amazon.com/iam/](https://console.aws.amazon.com/iam/)
2. En el menú izquierdo click en **Users**
3. Click **Create user**
4. **User name:** `ecommercenet-deploy`
5. Marca **Provide user access to the AWS Management Console** (opcional, por si necesitas entrar a la consola con este usuario)
6. **Console password:** Autogenerated
7. Click **Next**

### Asignar permisos:

8. Seleccionar **Attach policies directly** (tercera opción, a la derecha)
9. En el buscador de "Permissions policies" buscar y marcar estas 3 políticas:

| Búsqueda | Política a marcar |
|----------|------------------|
| `ElasticBeanstalk` | **AdministratorAccess-AWSElasticBeanstalk** |
| `S3Full` | **AmazonS3FullAccess** |
| `RDSFull` | **AmazonRDSFullAccess** |

> **Tip:** La lista tiene 1,481 políticas. Usa el buscador para filtrar — no intentes buscar manualmente.

10. Click **Next**
11. Revisar que aparezcan las 3 políticas + IAMUserChangePassword en el resumen
12. Click **Create user**

## 4. Crear Access Keys para CLI

Las Access Keys son como usuario/contraseña pero para la terminal.

### Pasos:

1. Después de crear el usuario, click en el nombre **ecommercenet-deploy** en la lista
2. Click en la tab **Security credentials**
3. Bajar a la sección "Access keys"
4. Click **Create access key**
5. Seleccionar **Command Line Interface (CLI)**
6. Marcar el checkbox de confirmación
7. Click **Next**
8. **Description tag:** `CLI EcommerceNet`
9. Click **Create access key**

### Copiar las claves:

```
Access Key ID:     AKIA................  ← copiar
Secret Access Key: xxxxxxxxxxxxxxxxxxxx  ← copiar (SOLO SE MUESTRA UNA VEZ)
```

> **⚠️ IMPORTANTE:** La Secret Access Key solo se muestra en este momento. Si la pierdes, tienes que borrar la key y crear una nueva. Guárdala en un lugar seguro (password manager, no en un archivo de texto).

## 5. Instalar AWS CLI en Windows

### Opción A: Con winget (recomendada)

```powershell
winget install Amazon.AWSCLI
```

### Opción B: Descarga directa

Si `winget` no funciona, descargar el instalador MSI desde:  
[https://awscli.amazonaws.com/AWSCLIV2.msi](https://awscli.amazonaws.com/AWSCLIV2.msi)

Instalar con doble click → Next → Next → Install → Finish.

### Verificar (CERRAR Y REABRIR PowerShell primero):

```powershell
aws --version
```

Deberías ver algo como:

```
aws-cli/2.34.27 Python/3.14.3 Windows/11 exe/AMD64
```

> **Error común:** Si después de instalar PowerShell sigue diciendo "aws is not recognized", cierra TODAS las ventanas de PowerShell/terminal y abre una nueva. Windows necesita recargar el PATH.

## 6. Instalar Python y EB CLI

EB CLI (Elastic Beanstalk Command Line Interface) requiere Python.

### Paso 1: Instalar Python

```powershell
winget install Python.Python.3.12
```

**Cerrar y reabrir PowerShell.** Verificar:

```powershell
python --version
# Python 3.12.10
```

### Paso 2: Instalar EB CLI con pip

```powershell
pip install awsebcli
```

Vas a ver muchas líneas de instalación. Al final puede aparecer un aviso:

```
[notice] A new release of pip is available: 25.0.1 -> 26.0.1
[notice] To update, run: python.exe -m pip install --upgrade pip
```

Esto es solo un aviso, no un error. Puedes ignorarlo o actualizar pip:

```powershell
python.exe -m pip install --upgrade pip
```

### Verificar:

```powershell
eb --version
```

Deberías ver:

```
EB CLI 3.27.1 (Python 3.12.10 (tags/v3.12.10:0cc8128, Apr  8 2025, 12:21:36) [MSC v.1943 64 bit (AMD64)])
```

## 7. Configurar credenciales con aws configure

Este paso conecta tu terminal con tu cuenta de AWS.

```powershell
aws configure
```

Te pregunta 4 cosas **una por una** (NO pegues todo de golpe):

```
AWS Access Key ID [None]: ← pegar tu Access Key ID y Enter
AWS Secret Access Key [None]: ← pegar tu Secret Access Key y Enter
Default region name [None]: us-east-1
Default output format [None]: json
```

> **Error que cometí y que debes evitar:**
> NO pegues las respuestas directamente en PowerShell antes de ejecutar `aws configure`.
> PowerShell interpreta el texto como comandos y falla con "The term 'AWS' is not recognized".
> Primero ejecuta `aws configure`, DESPUÉS responde cada pregunta cuando te la haga.

### ¿Por qué us-east-1?

Es la región más barata y con más servicios disponibles. DaCodes tiene sede en Houston, TX — us-east-1 (Virginia) está geográficamente cerca.

## 9. Errores comunes y soluciones

### Error: "aws is not recognized"

**Causa:** PowerShell no recargó el PATH después de la instalación.  
**Solución:** Cerrar TODAS las ventanas de PowerShell y abrir una nueva.

### Error: "The term 'AWS' is not recognized" al pegar credenciales

**Causa:** Pegaste las respuestas de `aws configure` directamente en PowerShell antes de ejecutar el comando.  
**Solución:** Primero ejecuta `aws configure`, luego responde cada pregunta cuando te la haga.

### Error: "pip is not recognized"

**Causa:** Python no está instalado o el PATH no se actualizó.  
**Solución:** `winget install Python.Python.3.12` → cerrar y reabrir PowerShell.

### Error: "eb is not recognized" después de `pip install awsebcli`

**Causa:** El directorio de scripts de Python no está en el PATH.  
**Solución:** Cerrar y reabrir PowerShell. Si persiste:

```powershell
python -m ebcli
```

### Aviso: "A new release of pip is available"

**Causa:** Tu versión de pip no es la más reciente.  
**Solución:** Es solo un aviso, no un error. Puedes ignorarlo o actualizar:

```powershell
python.exe -m pip install --upgrade pip
```

## 10. Seguridad — credenciales expuestas

### ¿Qué hacer si pegaste tus credenciales en un chat, email o repo público?

Esto me pasó durante el proceso — pegué las credenciales directamente en el chat de Claude en lugar de en `aws configure`.

**Pasos para remediar:**

1. Ve a [https://console.aws.amazon.com/iam/](https://console.aws.amazon.com/iam/)
2. Click en **Users** → **ecommercenet-deploy**
3. Tab **Security credentials**
4. En "Access keys" → encontrar la key comprometida
5. Click **Actions** → **Delete**
6. Crear una **nueva Access Key** (Paso 4 de esta guía)
7. Ejecutar `aws configure` de nuevo con las nuevas claves

> **Regla de oro:** Las Access Keys son como contraseñas. Nunca las compartas, nunca las pongas en código fuente, nunca las pegues en un chat público. Si se exponen, bórralas inmediatamente.

### Buenas prácticas:

- Usar **IAM users** para deploy, nunca la cuenta root
- Dar **permisos mínimos** necesarios (solo EB, S3, RDS — no AdministratorAccess)
- **Rotar las keys** cada 90 días
- Usar **environment variables** en producción, no archivos de configuración con claves
